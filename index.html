Why:

This app uses new browser features to establish direct connections between your browser and other visitors' browsers. One of those technologies is called Web Real-Time Communication (WebRTC). WebRTC is a cluster of technologies which enable two browsers to establish audio, video, or other streams directly by passing messages between a third party signaling server. Usually, the creator of the website would host that signaling server but if that server went down or was censored, then the app would cease to function.

This website uses a different browser api called web-push in the place of that signaling server. The Web Push api allows a website to use the signaling networks that browsers already have. Browser vendors use these servers to notify your browser when updates are available to download/install. They are also used to notify your browser of security certificate revocations in real-time. The browser manufacturers are adapting their signaling networks to support the web push protocol.

The Web Push specification specifies the browser api to get a subscription from those signaling networks as well as the message format which is always encrypted. In this app, instead of those signals (notifications) being send by a server, they are sent by other visitors' browsers and carry WebRTC signaling information. Both WebRTC and WebPush are encrypted end-to-end meaning that the signaling networks and servers used by WebRTC aren't able to intercept / hijack the connection between your browser and the person you're trying to talk with. However, it's important to understand that any privacy guarantees you have now are only one privacy policy change or one browser update away from being gone.
